# Task Evaluation Prompt

Проверь выполнение задачи в указанной директории и создай файл `result.md` с детальной оценкой.

## Входные данные
- Директория с результатами: `{task_directory}`

## Что нужно проверить

### 1. Корректность решения (Code Quality)
- Прочитай `TASK.md` — требования к задаче
- Найди и проверь созданные файлы (`.tsx`, `.ts`, `.css`, `.html` и др.)
- Оцени соответствие каждому требованию из TASK.md
- Проверь синтаксис и типы (для TypeScript)

### 2. Процесс выполнения (Execution Process)
- Прочитай `run_log.txt` — лог выполнения
- Прочитай `trajectory.json` — детальная траектория с токенами
- Оцени:
  - Сколько шагов потребовалось
  - Были ли ошибки и ретраи
  - Следовала ли модель инструкциям (первый шаг = `cat TASK.md`)
  - Были ли лишние/ненужные шаги
  - Пыталась ли модель выйти за пределы директории

## Формат result.md

```markdown
# Task {task_name} — Evaluation Result

## Overall Score: X/10 ✅/❌

---

## Requirements Checklist

| Requirement | Status | Notes |
|-------------|--------|-------|
| ... | ✅/❌ Pass/Fail | ... |

---

## Execution Process Analysis

### Trajectory Summary

| Metric | Value |
|--------|-------|
| Total API calls | X |
| Total steps | X |
| Exit status | Submitted/Failed/Timeout |
| Errors/Retries | X |

### Step-by-Step Breakdown

(Краткое описание каждого шага и его результата)

### Issues Found (если есть)

- Описание проблем в процессе выполнения

---

## Process Quality Assessment

### Efficiency Score: X/10

**Strengths:**
- ...

**Weaknesses:**
- ...

---

## Conclusion

(Краткий итог)
```

## Критерии оценки

### Оценка кода (0-10):
- 10: Все требования выполнены идеально
- 8-9: Выполнено с минимальными недочётами
- 6-7: Работает, но есть заметные проблемы
- 4-5: Частично работает
- 1-3: Серьёзные ошибки
- 0: Не работает / файлы не созданы

### Оценка процесса (0-10):
- 10: Оптимальный путь, 0 ошибок
- 8-9: Минимальные лишние шаги
- 6-7: Несколько ретраев, но справился
- 4-5: Много путаницы, но результат есть
- 1-3: Критические ошибки в процессе
- 0: Не смог завершить задачу

## Важно
- Сохрани `result.md` в ту же директорию с результатами задачи
- Будь объективен и конкретен в оценке
- Приводи примеры из кода/логов при описании проблем
