# Решение конфликта слияния в src/utils/pricing.ts

## Объединённая функциональность

Я объединил функциональность обеих веток, соблюдая все 6 бизнес-требований:

### 1. Интерфейс PricingInput
Объединил свойства из обеих веток:
- `basePrice` - базовая цена (общая для обеих веток)
- `discountPercent` - скидка (из feature/discount-system)
- `loyaltyPoints` - количество баллов для списания (из feature/loyalty-points)
- `userPointsBalance` - баланс баллов пользователя (из feature/loyalty-points)

### 2. Интерфейс PricingResult
Обновил return type для UI:
- `finalPrice` - итоговая цена после всех вычетов
- `pointsUsed` - количество списанных баллов

### 3. Логика функции calculateFinalPrice

**Шаг 1: Применение скидки**
- Скидка применяется к базовой цене (бизнес-требование #1)
- Скидка может быть 0-100%

**Шаг 2: Применение баллов лояльности**
- Баллы конвертируются 1:1 в рубли (бизнес-требование #4)
- Нельзя списать больше баллов, чем есть у пользователя (бизнес-требование #5)
- Нельзя списать больше 50% от цены после скидки (бизнес-требование #6)

**Шаг 3: Финальная цена**
- Цена не может быть меньше 0 (бизнес-требование #3)
